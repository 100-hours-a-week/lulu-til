# 한 줄 정리

---

## 2026-07-13

| 개념 | 정리 |
|------|------|
| **Kubernetes** | 컨테이너화된 애플리케이션을 어디서나 배포, 확장, 관리할 수 있는 오픈소스 시스템으로, 애플리케이션의 배포, 확장, 감시와 장애 복구를 자동화하여 운영 복잡성을 줄이기 위해 사용한다. |
| **선언적 관리** | 사용자가 작업 방법을 하나씩 명령하는 것이 아니라, 최종적으로 원하는 상태를 정의하는 방식이다. |
| **Master Node** | 클러스터 전체의 상태를 관리하고, 파드 배치 같은 것을 판단하는 중앙 제어 영역이다. |
| **etcd** | 클러스터의 설정, 상태를 저장하는 "분산" key-value 저장소이다. |
| **kube-scheduler** | 새로운 실행 단위(파드)를 어느 워커 노드에 배치할지 결정하는 구성 요소이다. |
| **kube-controller-manager** | 현재 상태를 원하는 상태로 맞추는 여러 제어 기능을 실행하는 구성 요소이다. |
| **Worker Node** | 실제 애플리케이션이 실행되는 서버로, 실제 작업을 수행하는 손발과 같은 존재이다. |
| **kubelet** | 각 워커 노드에서 애플리케이션 실행 상태를 관리하고, 컨트롤 플레인에 보고하는 에이전트이다. |
| **kube-proxy** | Worker Node에서 네트워크 트래픽 전달을 위한 규칙을 관리하는 구성 요소이다. |
| **CNI** | 컨테이너 네트워크를 설정하고 제거하는 방법을 정의한 표준 규격 인터페이스로, 쿠버네티스가 직접 모든 네트워크 환경을 지원하지 않기 때문에 사용한다. |
| **Flannel** | 여러 노드에 있는 Pod들이 서로 통신할 수 있도록 Pod 네트워크를 구성하는 네트워크 플러그인이다. |
| **VXLAN** | Pod에 있는 패킷을 Node 간 전달할 때, UDP 패킷 안에 한 번 더 감싸는 터널링 기술이다. |

## 2026-07-14

| 개념 | 정리 |
|------|------|
| **Pod** | 하나 이상의 컨테이너를 그룹화하여 실행 환경과 네트워크 자원, 스토리지를 공유하는 가장 작은 배포 단위로, 애플리케이션의 각 구성 요소를 논리적으로 묶어 단일 단위로 관리하고 배포할 수 있게 하기 위해 사용한다. |
| **Label** | 오브젝트에 부착되는 키-값 쌍으로, 오브젝트를 식별하고 그룹화하여 리소스 관리의 단순화와 유연성을 높이기 위해 사용한다. |
| **Selector** | 특정 레이블을 가진 오브젝트를 선택하고 필터링하는 메커니즘으로, 쿠버네티스 리소스를 Label 기반으로 동적으로 선택하고 관계를 관리하기 위해 사용한다. |

## 2026-07-15

| 개념 | 정리 |
|------|------|
| **Service** | 파드(Pod) 집합에 대한 네트워크 엔드포인트를 생성하고, 클러스터 내부 또는 외부에서 해당 파드들에 안정적으로 접근할 수 있도록 지원하는 쿠버네티스 리소스로, 동적으로 변경되는 파드의 IP 대신 고정된 네트워크 주소를 제공하기 위해 사용한다. |
| **ClusterIP** | 클러스터 내부에서만 접근 가능한 서비스 유형으로, 외부 네트워크에 노출되지 않으며 Service 생성 시 기본값으로 설정된다. 클러스터 내 구성 요소들이 서로 통신할 때 사용한다. |
| **NodePort** | 클러스터 외부에서 서비스에 접근할 수 있도록 클러스터 노드의 특정 포트를 할당하는 서비스 유형으로, 외부 네트워크에서 특정 노드 IP와 포트를 통해 서비스에 접근할 수 있게 한다. |
| **Liveness Probe** | 컨테이너의 생존 상태를 주기적으로 진단하는 메커니즘으로, 애플리케이션 실행 중 발생할 수 있는 비정상 상태를 자동으로 감지하고 컨테이너를 재시작하여 복구하기 위해 사용한다. |
| **Readiness Probe** | 컨테이너가 외부 요청을 처리할 수 있는지 상태를 주기적으로 진단하는 메커니즘으로, 준비되지 않은 컨테이너로 인한 서비스 장애를 예방하여 안정적이고 신뢰성 높은 서비스를 보장하기 위해 사용한다. |
| **Limit** | 컨테이너가 사용할 수 있는 최대한의 자원 한도이다. |
| **Request** | Pod를 실행하기 위해 쿠버네티스에게 요청하는 최소한의 자원이다. |

# 한 줄 정리

---

## 2026-06-29

| 개념 | 정리 |
|------|------|
| **DevOps** | 개발과 운영의 협업을 통해 소프트웨어 배포 속도와 안정성을 높이는 문화 및 방법론으로, 안정적인 배포/자동화/빠른 전달 등을 위해 사용된다. |
| **Docker** | 애플리케이션이 실행되는 데 필요한 모든 요소(코드, 라이브러리, 설정 등)를 하나로 묶어, 어떤 컴퓨터에서든 동일하게 실행되도록 도와주는 도구로, 애플리케이션의 실행 환경 차이의 문제를 좁히기 위해 사용한다. |
| **Namespace** | 컨테이너마다 프로세스, 네트워크, 파일시스템이 독립적으로 존재하는 것처럼 보이게 만드는 리눅스의 커널 기능으로, 하나의 전역 자원을 분리된 자원처럼 사용하기 위해 쓰인다. |
| **Cgroups** | 여러 프로세스를 하나의 그룹으로 묶고, 그 그룹의 CPU, 메모리, I/O, 프로세스 수 사용량을 제한하고 측정하는 리눅스 커널 기능으로, 자원을 제한하기 위해 사용한다. |
| **Dockerfile** | 도커 이미지를 생성하기 위한 명령어 및 설정이 담긴 텍스트 기반의 스크립트 파일로, 실행 환경을 수동 작업이 아니라 코드로 정의하기 위해 사용한다. |
| **Image** | 애플리케이션 실행에 필요한 파일을 여러 개의 읽기 전용 레이어로 쌓아 만든 실행 환경의 스냅샷으로, 공통 실행환경을 제공하기 위해 사용한다. |
| **OverlayFS** | 여러 개의 읽기 전용 이미지 레이어를 겹쳐, 애플리케이션이 하나의 완성된 파일시스템처럼 사용할 수 있게 하는 리눅스의 파일 시스템 기능이다. |
| **Container** | 도커 이미지를 바탕으로 실제 애플리케이션 프로세스를 실행한 격리된 실행 단위로, 실제 서비스를 실행하기 위해 사용한다. |
| **Registry** | 컨테이너 이미지를 저장하고 배포하기 위한 중앙 저장소로, 컨테이너 이미지를 공유하기 위해 사용한다. |

## 2026-06-30

| 개념 | 정리 |
|------|------|
| **Portainer** | Docker와 Kubernetes 같은 컨테이너 환경을 손쉽게 관리할 수 있도록 해주는 웹 기반 GUI 관리 도구로, 운영 가시성과 원격관리, 표준화를 위해 사용한다. |
| **Docker Socket** | 실제 컨테이너, 이미지, 볼륨, 네트워크를 실행하고 관리하는 엔진이다. |
| **RBAC** | 사용자의 직무나 '역할'을 기반으로 시스템, 네트워크, 데이터에 대한 접근 권한을 부여하는 보안 모델로, 대규모 조직에서 보안과 권한 관리를 효율적으로 자동화하기 위해 사용한다. |
| **Volume** | 컨테이너 간 데이터 공유 및 영속성을 제공하기 위해 호스트 파일 시스템에 생성되는 독립적인 저장소이다. |
| **Bind Mount** | 호스트 시스템(내 컴퓨터 또는 서버)의 특정 파일이나 디렉토리를 도커 컨테이너 내부의 경로에 직접 연결(마운트)하는 방식으로, 컨테이너와 호스트가 데이터를 실시간으로 공유하기 위해 사용한다. |

## 2026-07-01

| 개념 | 정리 |
|------|------|
| **Bridge Network** | 컨테이너들이 동일한 호스트 내에서 서로 통신할 수 있도록 연결해주는 도커의 가상 네트워크로, 컨테이너마다 독립적인 네트워크 환경을 만들고 필요한 컨테이너끼리만 같은 네트워크를 연결하기 위해 사용한다. |
| **Network Interface** | 컴퓨터나 네트워크 장비가 외부 네트워크와 데이터를 주고받기 위해 연결되는 물리적 또는 가상의 접점이다. |
| **Custom Bridge Network** | 도커 컨테이너들이 서로 격리된 환경에서 안전하게 통신할 수 있도록 사용자가 직접 정의하여 생성하는 가상의 네트워크로, 자동 DNS와 높은 격리 수준을 위해 사용한다. |
| **Docker Network Driver** | 도커 컨테이너가 서로 통신하거나 외부 네트워크(인터넷, 호스트 시스템 등)와 연결되는 방식을 정의하는 통신 모듈이다. |
| **Host Network** | 컨테이너가 별도의 네트워크를 만들지 않고 호스트의 네트워크 공간을 직접 공유하는 방식으로, 호스트에서 실행 중인 서비스가 로컬호스트로 직접 접근해야 하는 경우에 보통 사용한다. |
| **Overlay Network** | 여러 도커 호스트에 흩어져 있는 컨테이너를 하나의 가상 내부망처럼 연결하는 네트워크로, 여러 도커 호스트에 흩어져 있는 컨테이너를 하나의 가상 내부망처럼 연결하기 위해 사용한다. |

## 2026-07-02

| 개념 | 정리 |
|------|------|
| **Docker Compose** | 여러 컨테이너로 구성된 애플리케이션을 YAML 형식의 파일로 명확하게 정의하고, 단일 명령어로 동시에 실행 및 관리할 수 있도록 지원하는 도커의 공식 오픈소스 도구이다. |
| **IaC** | 서버, 네트워크, 데이터베이스 등 IT 인프라를 직접 수동으로 설치하고 설정하는 대신, 코드를 작성하여 인프라를 구축하고 관리하는 방식으로, 개발 및 운영 효율성을 극대화하기 위해 사용한다. |

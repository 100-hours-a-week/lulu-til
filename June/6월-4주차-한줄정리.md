# 한 줄 정리

---

## 2026-06-22

| 개념 | 정리 |
|------|------|
| **VPC** | 공용 클라우드 환경에서 사용자에게 독립된 가상 네트워크를 제공하는 서비스로, 서비스별 통신 경로와 접근 범위를 통제하기 위해 사용한다. |
| **퍼블릭 서브넷** | 서브넷 라우팅 테이블에 인터넷 게이트웨이 경로가 있는 서브넷으로 인터넷 연결이 필요한 대상을 위해 사용한다. |
| **프라이빗 서브넷** | 서브넷 라우팅 테이블에 인터넷 게이트웨이 경로가 없는 서브넷으로 보안을 위해 인터넷 연결을 막기 위해 사용한다. |
| **IAM** | AWS 리소스에 대한 사용자 인증과 권한 관리를 제공하는 서비스로, 작업자별로 인증과 권한을 통제하기 위해 사용한다. |
| **Security Group** | 클라우드 환경에서 네트워크 트래픽을 제어하는 가상의 방화벽으로, 서버를 인터넷에 무작정 노출하지 않고 필요한 대상에게만 통신을 허용하기 위해 사용한다. |
| **Route53** | AWS에서 제공하는 확장 가능한 클라우드 도메인 네임 시스템 서비스로, AWS에 있는 서비스와 유기적으로 연결되어 있는 도메인 서비스를 위해 사용한다. |
| **DNS** | 도메인을 어디로 연결할지 관리하는 서비스로 도메인별로 해당하는 IP로 호스팅하기 위해 사용한다. |
| **가상화** | 하나의 물리 컴퓨터를 여러 개의 독립된 가상 컴퓨터를 실행하는 기술로, 비용을 절감하고 자원을 효율적으로 사용하고 독립성, 안정성, 유연성, 확장성을 위해 사용한다. |
| **하이퍼바이저** | 물리 서버 한 대의 자원을 여러 개의 가상 컴퓨터가 나눠서 사용하고 서로를 격리해주는 소프트웨어 계층으로, CPU 분배, 메모리 분리, 격리성 등을 위해 사용한다. |
| **EC2** | AWS에서 제공하는 클라우드 가상 서버 서비스로, 초기 비용 및 시간을 절약하고 유연한 확장성을 위해 사용한다. |
| **EBS** | EC2에 연결되는 영구 블록 스토리지로, 독립적인 저장 공간을 위해 사용한다. |

## 2026-06-23

| 개념 | 정리 |
|------|------|
| **EFS** | 여러 EC2 인스턴스가 동시에 접근할 수 있는 확장 가능한 완전관리형 파일 스토리지 서비스로, 다수의 서버 간에 데이터를 실시간으로 공유하고 동기화하기 위해 사용한다. |
| **NFS** | 네트워크를 통해 다른 컴퓨터의 파일 시스템을 로컬에 있는 것처럼 공유하고 접근할 수 있게 해주는 프로토콜로, 원격 서버 간 효율적인 파일 공유 체계를 구축하기 위해 사용한다. |
| **Mount** | 특정 저장 장치나 원격 파일 시스템을 운영체제의 디렉터리 경로에 연결하는 과정으로, 외부 스토리지를 로컬 시스템의 일부처럼 인식하여 사용하기 위해 수행한다. |
| **S3** | 이미지, 동영상, 백업파일, 로그처럼 많은 파일을 객체 단위로 저장하고 인터넷 또는 AWS 서비스에서 접근할 수 있게 해주는 객체 스토리지 서비스로, 확장성과 높은 내구성 및 가용성 때문에 사용한다. |
| **Presigned URL** | 일정 시간동안 특정 S3 버킷, 객체에 접근할 수 있게 만든 서명된 URL로, 특정 객체에 시간 제한동안 업로드 혹은 다운로드 권한을 부여하기 위해 사용한다. |
| **RDS** | 클라우드에서 관계형 데이터베이스를 손쉽게 생성, 운영, 확장할 수 있도록 제공하는 완전관리형 서비스로, AWS에게 DB 운영환경을 대신 처리하도록 하기 위해 사용한다. |
| **Read Replica** | 주 데이터베이스(Primary)의 데이터를 실시간으로 복제한 읽기 전용 복제본으로, 읽기 작업(조회) 요청을 분산시켜 주 DB의 부하를 줄이고 애플리케이션 성능을 향상하기 위해 사용한다. |
| **Multi AZ** | 다른 AZ에 대기 DB를 유지하는 구성 방식으로, DB 장애 상황에서 서비스 중단을 줄이기 위해 사용한다. |

## 2026-06-24

| 개념 | 정리 |
|------|------|
| **ElastiCache** | Redis나 Memcached 같은 인메모리 캐시 서버를 직접 EC2에 설치하고 운영하지 않아도, AWS에서 간편하게 생성하고 사용할 수 있도록 제공하는 관리형 캐시 서비스로 빠른 반복 조회를 위해 사용한다. |
| **Cache-Aside** | 애플리케이션이 먼저 캐시를 조회하고, 없을 때만 DB를 조회한 뒤 캐시에 저장하는 방식이다. |
| **캐시 스탬피드** | 특정 캐시 데이터의 만료(TTL) 시점이나 삭제 순간에, 대규모 요청이 동시에 발생하여 데이터베이스(DB)나 원본 서버로 요청이 쇄도하고 이로 인해 서버가 과부하를 겪는 현상이다. |
| **Eviction** | 메모리 부족 상황을 방지하기 위해 중요도가 낮거나 가장 오래된 데이터를 지우는 정책(알고리즘)이다. |
| **ELB** | 사용자의 요청을 여러 서버에 나누어 전달하고, 장애 난 서버는 자동으로 제외하며 서비스가 느려지거나 멈추는 일을 줄여주는 AWS 관리형 로드 밸런서 서비스로, 서비스를 안정화하기 위해 사용한다. |
| **CloudFront** | 전 세계에 분산된 서버 네트워크(CDN)를 통해 사용자에게 콘텐츠를 빠르고 안전하게 전달하는 서비스로, 콘텐츠 캐싱과 글로벌 전송 가속을 위해 사용한다. |
| **CDN** | 사용자와 가장 가까운 곳에 위치한 분산 서버에 웹 콘텐츠(이미지, 동영상 등)를 미리 저장(캐싱)하여, 지리적 제약 없이 빠르고 안정적으로 콘텐츠를 제공하는 기술로, 속도 향상, 비용 절감, 안정성 등을 위해 사용한다. |

## 2026-06-25

| 개념 | 정리 |
|------|------|
| **Lambda** | 서버를 직접 관리하지 않고 코드만 실행할 수 있는 서버리스 컴퓨팅 서비스로, 비용 절감 및 확장성을 위해 사용한다. |
| **API Gateway** | 클라이언트와 백엔드 서비스 사이에서 API 요청을 관리/변환/보호해주는 완전관리형 서비스이다. |
| **CloudWatch** | AWS 리소스와 애플리케이션에서 발생하는 로그, 지표, 이벤트를 수집/모니터링하는 통합 관제 서비스이다. |
| **Observability** | 시스템 내부의 상태를 외부에서 수집 가능한 데이터를 통해 추론하고 이해하는 능력이다. |

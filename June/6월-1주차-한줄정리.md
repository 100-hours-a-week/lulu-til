# 한줄 정리

---

## 2026-06-01

| 개념 | 정리 |
|------|------|
| **ORM** | 자바의 객체와 데이터베이스 테이블을 자동으로 연결하고 변환해, 객체 지향 코드로 데이터를 조작할 수 있게 해주는 기술로 객체와 데이터베이스의 중간 다리 역할이다. 개발자가 SQL문이 아니라 비즈니스 로직에만 집중할 수 있기 때문에 사용한다. |
| **JPA** | 자바의 표준 ORM 인터페이스이다. 개발자가 직접 JDBC를 다루지 않고, 중간에서 ORM의 역할을 수행시키기 위해 사용한다. |
| **Entity** | 데이터베이스 테이블과 1:1로 매핑되어 데이터를 객체 형태로 표현하는 자바 클래스이다. 데이터와 규칙을 한 곳에 응집시켜 수정을 용이하게 하기 위해 사용한다. |
| **영속성 컨텍스트** | JPA가 엔티티 객체를 관리하며, 동일 트랜잭션 내에서 변경사항을 추적하고 자동으로 데이터베이스와 동기화하는 메모리 상의 저장소이다. JPA의 성능을 높이고 엔티티 관리를 실시간으로 하기 위해 사용한다. |
| **연관관계 매핑** | 엔티티 객체 간의 관계를 설정하여, 객체지향적으로 연결된 데이터 구조를 정의하고 관리할 수 있도록 해주는 매핑 구조이다. 데이터 정합성과 설계 일관성을 유지하기 위해 사용한다. |
| **Proxy** | JPA에서 지연 로딩을 사용할 때, 실제 엔티티 대신 먼저 생성되어 전달되는 가짜 객체이다. 지연 로딩을 통해 성능을 높이기 위해 사용한다. |

## 2026-06-02

| 개념 | 정리 |
|------|------|
| **값 타입** | JPA의 3가지 값 타입으로 기본값, 임베디드, 컬렉션 값 타입이 있다. 코드 재사용성과 객체 지향적인 개발을 위해 다양한 타입이 존재한다. |
| **임베디드 매핑** | 두 개 이상의 값을 하나의 객체로 묶은 임베디드 타입을 엔티티에 매핑하는 것이다. 코드 재사용성을 높이고 객체 지향적인 개발을 위해 사용한다. |
| **JPQL** | 엔티티 객체를 대상으로 데이터를 조회하거나 조작할 수 있도록 제공되는 객체 지향 쿼리 언어이다. 복잡한 관계식, 조인, 조건 등을 처리하기 위해 사용한다. |
| **Spring Data JPA** | JPA 기반의 데이터 처리를 자동화해 인터페이스만으로 리포지토리를 쉽게 구현할 수 있도록 도와주는 Spring 모듈이다. 개발 생산성과 유지 보수 효율을 높이기 위해 사용한다. |
| **QueryDSL** | 자바 코드로 JPQL이나 SQL을 직접 작성하지 않고, 메서드 호출과 객체 지향 문법을 이용해 쿼리를 표현할 수 있도록 해주는 쿼리 라이브러리이다. 동적 쿼리를 최적화하고 문법 오류를 발견하기 쉽고 코드 재사용성을 높여줘서 사용한다. |

## 2026-06-04

| 개념 | 정리 |
|------|------|
| **필터** | HTTP 요청이 디스패처 서블릿에 도달하기 전후로 웹 애플리케이션 수준에서 요청/응답을 가로채는 컴포넌트이며, 주로 인코딩 설정, 보안(인증/인가), 로깅 등 웹 전반에 공통되는 전처리 작업을 수행하기 위해 사용한다. |
| **인터셉터** | 디스패처 서블릿과 컨트롤러 사이에서 요청을 가로채는 스프링 MVC 제공 컴포넌트이며, 로그인 체크, 권한 검증, 컨트롤러 실행 시간 측정 등 스프링 컨텍스트(빈)를 활용한 세밀한 비즈니스 로직 전후 처리를 위해 사용한다. |
| **ExceptionHandler** | 특정 컨트롤러 내부에서 발생한 예외를 감지하고 매핑된 메서드로 가로채어 처리하는 기능이며, 비즈니스 로직과 예외 처리 코드를 분리하여 특정 컨트롤러에 특화된 에러 응답을 깔끔하게 제어하기 위해 사용한다. |
| **RestControllerAdvice** | 여러 컨트롤러에서 발생하는 예외를 한곳에서 전역적으로 가로채어 처리할 수 있게 해주는 애플리케이션 단위의 컴포넌트이며, 애플리케이션 전역의 에러 응답 포맷을 일관되게 유지하고 예외 처리 코드를 중앙 집약적으로 관리하기 위해 사용한다. |
| **ResponseStatus** | 에러 클래스나 예외 처리 메서드에 원하는 HTTP 상태 코드를 직접 지정할 수 있게 해주는 어노테이션이며, 별도의 응답 객체를 복잡하게 생성하지 않고도 클라이언트에게 직관적이고 정확한 HTTP 상태 코드를 간결하게 반환하기 위해 사용한다. |
| **파일 처리** | 클라이언트가 대용량 파일이나 이미지를 HTTP 요청(Multipart)에 담아 보낼 때 이를 스프링 내부 객체로 파싱하고 다루는 기능이며, 프로필 사진 업로드, 문서 첨부 등 바이너리 데이터를 안전하고 효율적으로 서버에 저장 및 관리하기 위해 사용한다. |
| **고아 이미지** | 데이터베이스의 게시글이나 상품 정보는 삭제되었으나, 실제 스토리지(서버 디렉토리, AWS S3 등)에는 지워지지 않고 남아있는 파일들을 뜻하며, 불필요한 스토리지 비용 낭비를 막고 서버 용량을 효율적으로 관리하기 위해 배치 작업 등으로 추적하여 삭제해야 한다. |

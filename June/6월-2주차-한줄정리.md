# 한줄 정리

---

## 2026-06-08

| 개념 | 정리 |
|------|------|
| **운영체제** | 하드웨어와 소프트웨어를 관리하며, 사용자와 컴퓨터 간 상호작용을 가능하게 하는 시스템 소프트웨어 |
| **Kernel** | 운영 체제가 실행되는 환경 |
| **System Call** | 운영체제의 커널 기능을 요청하는 공식 통로(인터페이스) |
| **Virtual Memory** | 물리적인 RAM보다 큰 프로그램을 실행하면서, 동시에 여러 개의 프로세스를 안전하게 실행하려고 하는 것 |
| **Swap** | 사용하지 않는 데이터를 디스크의 스왑 공간으로 이동하여 RAM 메모리를 확보하고, 필요할 때 다시 메모리로 복구하는 방식 |
| **Paging** | 메모리를 동일한 크기의 페이지로 나누고, 프로그램도 동일한 크기의 페이지로 나눠 메모리에 할당하는 방식 |
| **Segmentation** | 프로그램을 논리적 단위(코드, 데이터, 스택 등)로 나누어 메모리에 연속된 공간을 찾아 데이터를 할당하는 방식 |
| **Internal Fragmentation** | 할당된 메모리 블록보다 프로세스가 필요한 공간이 작아서 블록 안에 버려지는 빈 메모리 공간 |
| **External Fragmentation** | 프로세스들이 들어갔다 나왔다 반복하며 메모리 중간중간에 작은 빈 공간들이 듬성듬성하게 남아서, 전체 공간은 충분한데 연속된 공간이 아니라서 새로운 큰 프로세스를 못 넣는 것 |
| **Process** | 실행 중인 프로그램의 흐름 |
| **Thread** | 프로세스 내에서 병렬 처리를 위해 생성되는 실행 흐름의 최소 단위 |
| **IPC** | 프로세스 간의 통신 |
| **PCB** | 운영체제가 각 프로세스를 효율적으로 관리하기 위해 모든 실행 정보를 담아두는 핵심 자료 구조 |

## 2026-06-09

| 개념 | 정리 |
|------|------|
| **동시성** | 하나의 CPU가 작업을 빠르게 전환하여 마치 여러 작업이 동시에 실행되는 것처럼 보이게 하는 방식으로 자원을 효율적으로 관리하고 최적화하기 위해 사용 |
| **스케줄러** | 운영체제의 핵심 구성 요소로, CPU와 같은 시스템 자원을 적절히 분배하기 위해 프로세스나 스레드의 실행 순서를 관리하는 역할로 CPU 이용률을 최대화하고 공정한 분배를 위해 사용 |
| **Context Switching** | 기존 프로세스 작업에서 새로운 프로세스로 넘어가는 것 |
| **병렬성** | 여러 CPU 코어 또는 다중 프로세서를 사용하여 여러 작업을 실제로 동시에 실행하는 개념으로 시스템 성능을 극대화하기 위해 사용 |
| **Job Queue** | 하드디스크(보조기억장치)에 있는 프로그램이 실행되기 위해 메모리(주기억장치)에 올라가기 전, 대기하는 프로그램(Job)들이 모여 있는 저장 공간으로 순서를 관리하고 제어하기 위해 사용 |
| **Ready Queue** | 메모리에 성공적으로 로드되어, CPU를 할당받아 실제로 실행되기만을 기다리고 있는 프로세스들의 대기줄로 당장 사용 가능한 프로세스를 따로 모아 관리하기 위해 사용 |

## 2026-06-10

| 개념 | 정리 |
|------|------|
| **공유 자원** | 여러 사용자나 프로세스, 스레드가 동시에 접근하거나 사용하는 시스템 내의 자원으로 메모리를 절약하고 데이터 일관성을 유지하기 위해 사용한다. |
| **임계 영역** | 공유 자원에 접근하는 프로그램 코드의 일부분으로 데이터 무결성을 유지하고 시스템 안정성을 확보하기 위해 사용한다. |
| **동기화** | 여러 스레드가 데이터를 안전하게 읽고, 쓸 수 있도록 접근 순서를 제어하는 기술 |
| **락** | 공유 자원 앞에 걸어두는 자물쇠. 임계 영역에 한 번에 하나의 스레드만 들어가게 하는 도구로 공유 자원을 안전하게 사용하기 위해 사용한다. |

## 2026-06-11

| 개념 | 정리 |
|------|------|
| **교착상태** | 서로가 가진 자원을 내놓기를 기다리며 둘 이상의 프로세스가 영원히 멈춰버리는 현상 |
| **기아상태** | 우선순위가 밀려 특정 프로세스가 원하는 자원을 영영 할당받지 못하고 굶주리는 현상 |
| **세마포어** | 공유 자원에 접근할 수 있는 허용 개수를 두어, 여러 프로세스가 동시에 자원을 안전하게 쓰도록 제어하는 기법 |

# 한줄 정리

---

## 2026-06-15

| 개념 | 정리 |
|------|------|
| **Network** | 데이터를 송수신하는 기기들 간의 연결을 통해 정보와 자원을 공유할 수 있게 하는 시스템으로 정보/자원 공유, 원격 접속, 서비스 제공 등을 위해 사용한다. |
| **NIC** | 컴퓨터, 서버 또는 기타 네트워크 장치가 네트워크에 연결되도록 해주는 하드웨어 구성요소로 호스트와 네트워크 사이에서 데이터를 주고 받기 위해 사용한다. |
| **MAC Address** | 네트워크 장치의 물리적 주소를 나타내는 고유한 식별자로 같은 네트워크 안에서 장치를 구분하기 위해 사용한다. |
| **Gateway** | 서로 다른 네트워크를 연결하여 통신을 가능하게 하는 장치로 서로 다른 네트워크를 연결하고 데이터를 변환/필터링 하기 위해 사용한다. |
| **Router** | 서로 다른 네트워크를 연결하고, IP 주소를 보고 패킷이 갈 다음 경로를 결정하는 장비로 엄청나게 쪼개져 있는 네트워크에서 목적지를 지정해주기 위해 사용한다. |
| **Switch** | 같은 네트워크 안에서 MAC 주소를 보고 데이터를 필요한 장치에게만 전달하는 장비로 같은 네트워크 안에서 장치를 연결하기 위해 사용한다. |

## 2026-06-16

| 개념 | 정리 |
|------|------|
| **Inline** | 보안 장비나 네트워크 장비를 실제 데이터 트래픽이 지나가는 주 경로에 직접 직렬로 연결하여, 트래픽을 실시간으로 감시하고 차단하기 위해 사용한다. |
| **Out of Path** | 메인 네트워크 경로 밖에서 트래픽을 복사(Mirroring)하여 분석하는 구성 방식으로, 네트워크 성능에 영향을 주지 않고 모니터링하기 위해 사용한다. |
| **WireShark** | 네트워크 패킷을 실시간으로 캡처하고 프로토콜 단위로 상세히 분석할 수 있게 해주는 오픈소스 소프트웨어 툴이다. |
| **Failover** | 주 시스템이나 장비에 장애가 발생했을 때, 준비되어 있던 예비(백업) 시스템으로 자동으로 전환하여 서비스가 중단 없이 유지되도록 하는 기능이다. |
| **SPOF (Single Point of Failure)** | 시스템의 구성 요소 중 하나가 고장 나면 전체 시스템이 마비되어 버리는 단일 실패 지점을 의미한다. |
| **네트워크 캡처** | 네트워크 상에서 오고 가는 패킷(데이터)을 수집하고 저장하여 분석, 문제 해결, 보안 점검 등에 활용하는 행위이다. |

## 2026-06-17

| 개념 | 정리 |
|------|------|
| **Proxy** | 클라이언트와 서버 사이에서 대리인 역할을 수행하여 보안, 속도 향상, 익명성 보장 등을 제공하는 시스템이다. |
| **Forward Proxy** | 클라이언트 측에 위치하여 클라이언트의 요청을 대신 서버에 전달해 주는 프록시로, 내부 사용자의 외부 접근 통제 및 캐싱을 위해 사용한다. |
| **Reverse Proxy** | 서버 측 전면에 위치하여 외부 클라이언트의 요청을 받아 내부 서버로 분산해 주는 프록시로, 실제 서버 보안 및 로드 밸런싱을 위해 사용한다. |
| **VPN (Virtual Private Network)** | 공용 네트워크(인터넷)를 마치 전용선처럼 안전하게 사용할 수 있게 해주는 가상 사설망으로, 암호화된 터널을 통해 원격지 간 안전한 데이터 송수신을 위해 사용한다. |
| **NAT (Network Address Translation)** | 사설 IP 주소를 공인 IP 주소로(또는 그 반대로) 변환해 주는 기술로, 제한된 공인 IP를 절약하고 내부 네트워크를 보호하기 위해 사용한다. |

### 🚀 Today I Learned

#### | 상속

`상속`이란, 부모 클래스의 속성과 메서드를 자식 클래스가 물려받는 것

**헷갈리는 상속 포인트**

- 클래스 관계가 분류라면 상속을 쓴다
  ex) 무기 - 검, 활, 도끼
- 조합이라면 필드, 속성을 추가한다
  ex) 아메리카노, 아이스아메리카노(아메리카노+얼음)


#### 다형성

`다형성`이란, 하나의 타입으로 여러 객체를 다룰 수 있게 하는 객체지향 특성


**다형성-오버로딩**

- 하나의 클래스 안에서 같은 이름의 메서드를 매개변수의 개수나 타입을 다르게 정의하는 것
- 메서드, 생성자에 적용 가능
--> 코드의 일관성과 가독성을 높일 수 있다

```
  public void change();
  public void change(int i);
  public void change(String s, int i);

```

**다형성-오버라이딩**

- 부모클래스의 메서드를 자식클래스에서 재정의하는 것
- 자식클래스에서 *메서드의 이름, 매개변수, 리턴타입*을 유지하면서 본문만 바꿔 정의하는 것

```
public class Car {
  void drive(int speed){
    System.out.println(speed);
  }
}

public class Bus extends Car {
  @override
  void drive(int speed){
    if(speed > 70){
      System.out.println("속도 초과!");
    } else {
      System.out.println(speed);
    }
  }
}

```

#### 캡슐화

`캡슐화`란 객체의 내부 상태를 외부에서 직접 접근하지 못하도록 감추고, 메서드를 통해 간접적으로 접근하도록 하는 객체지향 개념

- 클래스 내부의 필드(변수)를 `private`으로 선언하여 외부에서 접근하지 못하도록 감추고, public 메서드를 통해 제한적으로
  접근하거나 수정할 수 있도록 구성
- 사용 이유 : 객체의 내부 구현은 보호하고, 데이터의 무분별한 접근이나 수정을 방지하기 위해서


#### Interface

`Interface`란 구현 세부사항없이 클래스가 구현해야 할 메서드의 집합만을 정의한 설계 명세서

- 클래스 간에 "이 기능을 반드시 제공해야 한다"는 명시적 요구사항의 집합이며, 이를 통헤 어떤 기능이 제공될 것인지를 정해놓는다
- 인터페이스는 메서드의 이름과 매개변수만을 정의하고, 실제 동작은 포함하지 않음
  구현은 오직 해당 인터페이스를 따르는 클래스에서 이루어진다 (선언된 모든 메서드를 반드시 오버라이딩하여 구현)
- 사용 이유 : 기능이 미완성된 설계 단계에서 구조를 먼저 정의하고, 구현자 간 역할을 분리하여 유연하고 확장 가능한 개발을 위해

```
public interface Car {
  void startEngine();
}

public class ElectricCar implements Car {
  @Override
  public void startEngine(){
    System.out.println("부릉");
  }
}

```

그렇다면
`인터페이스랑 상속이 뭐가 다른거지?`

| **구분** | **상속 (Inheritance)** | **인터페이스 (Interface)** |
| --- | --- | --- |
| **핵심 키워드** | **IS-A** (~는 ~이다) | **CAN-DO** (~를 할 수 있다) |
| **관계** | 수직적, 계층적 구조 | 수평적, 기능적 확장 |
| **목적** | 부모의 기능을 물려받아 **재사용** | 클래스에 특정 **자격/능력 부여** |
| **제한** | 단일 상속만 가능 (부모는 하나) | **다중 구현 가능** (자격증은 여러 개) |
| **비유** | "생물학적 혈통" (포유류 → 고양이) | "전문 자격증" (운전 가능, 수영 가능) |


#### Generic

`Generic`란 데이터 타입을 클래스나 메서드가 사용할 때까지 지정하지 않고, 나중에 지정할 수 있도록 해주는 타입 매개변수 문법

- 특정 자료형에 의존하지 않고, 타입을 정하지 않은 채로 두고, 객체를 생성하거나 메서드를 호출할 때 구체적인 타입을 지정

```
public class Garage<T> {
  private T car;

  public Garage(T car){
    this.car = car;
  }
}

public class Main {
  public static void main(String[] args){
    Garage<String> garage1 = new Garage<>("전기차");
    Garage<Integer> garage2 = new Garage<>(2026);
  }
}

```

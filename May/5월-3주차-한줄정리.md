## 📝 한줄 정리

---

### 📆 2026-05-26

| 용어 | 정리 |
|---|---|
| **데이터베이스** | 데이터의 집합으로 정보를 효율적으로 안전하게 저장하고, 검색/저장/수정/삭제하기 위해 사용 |
| **ERD** | 개체와 관계를 한 눈에 알아볼 수 있도록 그려놓은 도표로, DB에서 테이블들과 그들의 관계를 보기 쉽게 정리하기 위해 사용 |
| **Index** | 데이터베이스에 있어 테이블에 대한 조회 동작의 속도를 높여주는 자료구조로, 검색 속도를 높이기 위해 사용 |
| **Tree** | 노드로 이뤄진 계층적인 자료 구조로, 탐색 속도를 높이기 위해 사용 |
| **Hash** | 키 값을 hash function을 이용해 특정 출력값으로 변경하는 기술로, 단일 검색의 속도를 높이기 위해 사용 |
| **Full Text Index** | 텍스트 검색에서 컴퓨터에 저장된 문서 또는 전문 데이터베이스를 검색하는 기법으로, 문장 단위 탐색의 속도를 높이기 위해 사용 |
| **Transaction** | DBMS에서 한번에 수행할 연산의 단위로, 데이터베이스의 무결성과 일관성을 보장하고 여러 작업을 하나의 논리적인 작업 단위로 묶어 처리하기 위해 사용 |

### 📆 2026-05-27

| 용어 | 정리 |
|---|---|
| **HTTP 자격 증명 헤더** | HTTP는 Stateless하기 때문에 서버는 클라이언트가 누구인지 기억하지 못하므로, 보호된 리소스에 접근할 때 매 요청마다 신원을 증명하기 위해 사용하는 헤더 |
| **Basic Authentication** | 가장 단순한 HTTP 인증 방식으로, 별도의 로그인 폼 없이 HTTP Header에 사용자의 ID와 PW를 인코딩하여 서버에 전송하는 방식 |
| **Bearer Token** | OAuth 프레임워크에서 액세스 토큰으로 사용하는 토큰의 유형으로, 불투명한 문자열 |
| **JWT** | 당사자 간에 정보를 JSON 객체로 안전하고 가볍게 전송하기 위한 웹 표준 개방형 규격. 마치 홀로그램 인쇄 티켓처럼 내부 내용과 발급자의 서명이 함께 담겨 있다 |
| **Access Token** | 실제 API 리소스에 접근하기 위한 토큰으로, 출입증의 역할 |
| **Refresh Token** | Access Token을 새로 발급받기 위한 토큰으로, 인증서의 역할 |
| **RTR** | Refresh Token이 탈취되는 것을 막기 위한 전략으로, 기존의 Refresh Token을 사용할 때마다 새로운 Refresh Token을 발급하고 이전 토큰은 즉시 폐기하는 보안 기법 |

### 📆 2026-05-28

| 용어 | 정리 |
|---|---|
| **Key-Value Database** | 단순한 키-값 한 쌍으로 데이터를 저장하는 방식으로, 빠른 속도를 위해 사용 |
| **Document Database** | JSON, XML 같은 문서(Document) 단위로 데이터를 저장하는 방식 |
| **Column-oriented Database** | 테이블 구조처럼 보이지만, 행(Row)이 아닌 컬럼(Column) 단위로 데이터를 저장하는 방식 |
| **Graph Database** | 노드(Node)와 엣지(Edge) 구조로 데이터를 저장하는 방식으로, 관계(Relationship) 중심 |
| **Cache** | 자주 사용되는 데이터를 임시로 저장해두고, 동일한 요청이 오면 저장된 결과를 바로 반환하는 기법 |

### 📆 2026-05-29

| 용어 | 정리 |
|---|---|
| **동시성 제어** | 동시에 데이터에 접근할 때 꼬이지 않도록 순서를 제어하여 데이터의 일관성과 무결성을 유지하기 위해 사용 |
| **낙관적 락** | 충돌이 적을 거라 예상하고 락 대신 버전 관리로 검증하여 트랜잭션 대기 없는 높은 성능을 확보하기 위해 사용 |
| **비관적 락** | 충돌이 잦을 거라 예상하고 DB에 직접 락을 걸어 결제/송금 등 극도의 데이터 안전성을 보장하기 위해 사용 |
| **MVCC** | 데이터의 다중 버전을 관리해 읽기와 쓰기가 서로를 막지 않게 하여 DB의 동시 처리 성능을 극대화하기 위해 사용 |
| **OAuth** | 비밀번호 없이 외부 서비스(구글/카카오 등)의 권한을 위임받아 편리한 소셜 로그인과 높은 보안성을 동시에 달성하기 위해 사용 |

클래스 : 객체를 생성하기 위해 속성과 기능을 묶어, 코드의 재사용성과 유지보수성을 높여주는 템플릿
메서드 : 클래스에 종속되어 객체가 수행하는 기능을 정의한 코드 블록
생성자 : 객체가 생성될 때, 자동으로 호출되는 초기 작업을 위한 메서드
객체 : 클래스를 기반으로 생성된 실체
상속 : 부모 클래스의 속성과 메서드를 자식 클래스가 물려받는 것

2026-05-18
Web : 인터넷상에서 정보를 보고 요청하고 다양한 서비스를 제공받을 수 있는 시스템 
클라이언트 : 서비스를 Request 하는 단말기 또는 소프트웨어
서버 : 비즈니스 로직을 처리하여 클라이언트에게 서비스나 자원을 제공하는 컴퓨터 시스템이나 프로그램
HTTP : 클라이언트와 웹서버가 어떻게 통신할지를 표준화한 규약
HTTP Mesasge : 클라이언트와 서버 간 파일 등을 주고 받는 통신의 기본 단위
HTTP Method : 클라이언트가 서버한테 요청할 때 어떤 요청을 하는지 명시하기 위해 구분된 것
HTTP Status Code : 클라이언트가 서버한테 요청할 때, 서버가 클라이언트에게 통신 결과를 명확하게 전달하기 위해 사용하는 코드값
HTTP URL : 리소스의 위치와 종류를 나타내는 주소
Query String : 클라이언트가 웹 페이지에 대한 추가 정보를 서버에 전달하기 위해 URL 끝에 붙이는 문자열로 무언가를 정렬, 필터링할 때 사용
Path Variable : URL 경로 자체에 변수 값을 포함시켜, 특정 리소스를 식별하는 변수
RESTful API  : URI는 자원을 표현하고 자원에 대한 행위는 HTTP 메서드로 표현한다는 규칙을 지키는 API로 표준화된 규칙으로 일관되게 통신하기 위해 사용한다
스프링 : 자바 애플리케이션에서 객체 생성과 의존성 관리를 자동화해 구조를 단순하고 유연하게 만드는 프레임워크로 개발 생산성과 유지보수성을 높이기 위해 사용한다
IoC  : 객체의 생성, 주입 등 생명 주기를 개발자가 제어하지 않고 스프링 컨테이너가 관리한다는 개념으로 제어권을 역전시켜 클래스간 결합도를 낮추고 유지보수성을 높이기 위해 사용한다
DI  : 객체가 필요한 다른 객체(의존)를 직접 만들지 않고, 외부 컨테이너에서 만들어서 주입해주는 방식으로 재사용성과 유지보수성을 높이기 위해 사용한다
AOP : 공통 관심사와 핵심 관심사를 분리하는 방식으로 모든 서비스에서 반복되는 관심사를 묶어 중복되는 부분을 간편하게 통일시키기 위해 사용한다
Spring bean  : 스프링 컨테이너가 직접 생성하고 관리까지 책임지는 객체로 사용자에게 주어진 책임 줄이기 위해 사용한다
빈 컨테이너:  빈들을 생성하고 관리하고 주입하는 IoC/DI 엔진으로  개발자의 책임을 줄이고 유지보수성을 높이기 위해 사용된다

Spring Web MVC : DispatcherServlet 을 중심으로 요청을 컨트롤러에 분배하고, 컨트롤러가 처리 결과를 Model에 담아 뷰로 전달하는 프레임워크
DispatcherServlet : 모든 요청을 가장 먼저 수신하여 적절한 컨트롤러로 연결하고, 결과를 View로 레더링해 응답하는 핵심. 서블릿
대칭키 : 하나의 키로 암호화, 복호화를 하는 것
비대칭키 : 공개키, 개인키로 암호화, 복호화를 하는 것
HTTPS : HTTP에 SSL/TLS를 적용해 웹에서 HTTP로 전송되는 데이터를 암호화하고, 서버의 신원을 확인해 통신을 보호하는 보안 프로토콜

OOP : 프로그램을 단순히 명령어의 모음이 아니라, 독립된 단위인 '객체'들의 상호작용으로 파악하여 작성하는 프로그래밍

JVM : 자바 바이트코드(.class)를 컴파일하여 해당 OS가 이해할 수 있는 기계어로 실행해 주는 가상 컴퓨터

Stateless : 서버가 클라이언트의 이전 상태(과거의 요청 내역 등)를 전혀 보존하지 않는 독립적인 통신 방식
Connectionless : 클라이언트가 요청을 보내고 서버가 응답을 마치면 즉시 연결(Connection)을 끊어버리는 성질
인증: 사용자가 주장하는 그 사람(신원)이 정말 맞는지 확인하는 과정(예: 로그인)
인가: 인증된 사용자가 특정 자원에 접근하거나 특정 권한(예: 관리자 기능 등)을 가질 수 있는지 허용 여부를 검증하는 과정
쿠키: 서버가 클라이언트의 웹 브라우저에 저장하는 작은 텍스트 데이터 조각
세션: 클라이언트의 상태 정보를 브라우저가 아닌 보안이 철저한 서버 측 메모리에 저장하고 관리하는 기술
프레임워크: 애플리케이션 개발을 위한 전체적인 구조와 흐름을 이미 제공하며, 개발자는 그 틀 안에서 코드를 채워 넣는 기본 뼈대
라이브러리: 개발자가 특정 기능을 수행하기 위해 필요할 때마다 부품처럼 호출해서 사용하는 유용한 코드(도구)의 집합
API: 서로 다른 소프트웨어가 규칙에 따라 데이터를 주고받을 수 있도록 연결해 주는 창구이자 명세서
